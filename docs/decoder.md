# Decoder Task

## Ответственный

Николай

## Цель

Реализовать слой декодирования, который восстанавливает исходный секрет из набора `shares`.

## Контракт

```python
recover_secret(shares: list[str]) -> str
```

## Что приходит на вход

- `shares: list[str]`

## Что должно уйти на выход

- `str`

## Обязательные проверки

- список `shares` не пустой
- каждый `share` валиден по формату
- `shares` согласованы между собой
- `shares` достаточно для восстановления

## Ошибки

- `malformed share`
- `inconsistent shares`
- `insufficient shares`

Backend дальше сам конвертирует эти ошибки в свой HTTP-формат.

## Что входит в задачу

- разбор строкового формата `share`
- проверка согласованности метаданных
- восстановление секрета по достаточному набору `shares`
- возврат результата как `str`

## Критерий готовности

- decoder принимает `shares` от encoder без дополнительной адаптации
- при достаточном наборе `shares` возвращается исходная строка
- при ошибках возвращается понятное исключение

## Интеграция с backend

Backend передает в decoder только:

- `shares: list[str]`

Decoder возвращает только:

- `secret: str`

## Полезная практика DNP

- `strict input validation`
- `well-defined failure modes`
- `loose coupling`: decoder не знает про transport-слой
